# Microsoft Defender XDR 自動化機能まとめ

## 概要

Microsoft Defender XDR は、セキュリティオペレーションチームの負荷を軽減し、脅威への対応を迅速化するための複数の自動化機能を提供する。主な自動化機能は以下の 4 つに大別される。

1. **自動調査と応答 (AIR: Automated Investigation and Response)**
2. **自動攻撃中断 (Automatic Attack Disruption)**
3. **カスタム検出ルール (Custom Detection Rules)**
4. **アクションセンター (Action Center)**

---

## 1. 自動調査と応答 (AIR)

### 概要

セキュリティアラートがトリガーされると、自動調査が開始され、脅威の判定と修復アクションが自動的に実行される。仮想アナリストとして 24 時間 365 日稼働し、Tier 1/Tier 2 のセキュリティアナリストが行う調査・修復手順を模倣する。

### 調査プロセス

1. アラートがインシデントを生成し、自動調査が開始される
2. 各証拠（エビデンス）に対して判定（Verdict）が下される
3. 判定に基づき修復アクションが実行される

### 判定の種類

| 判定 | 説明 |
|---|---|
| **Malicious（悪意あり）** | 脅威が確認されたエンティティ |
| **Suspicious（疑わしい）** | 疑わしい動作が検出されたエンティティ |
| **No threats found（脅威なし）** | 脅威が検出されなかったエンティティ |

### 対象となるサービスとエンティティ

| エンティティ | 保護サービス |
|---|---|
| デバイス（エンドポイント） | Microsoft Defender for Endpoint |
| オンプレミス Active Directory ユーザー、エンティティの動作とアクティビティ | Microsoft Defender for Identity |
| メールコンテンツ（ファイルや URL を含むメール） | Microsoft Defender for Office 365 |

### 修復アクション一覧

| デバイス（エンドポイント）の修復アクション | メールの修復アクション | ユーザー（アカウント）の修復アクション |
|---|---|---|
| 調査パッケージの収集 | URL のブロック（クリック時） | ユーザーの無効化 |
| デバイスの分離（元に戻すことが可能） | メールメッセージまたはクラスターの論理削除 | ユーザーパスワードのリセット |
| マシンのオフボード | メールの検疫 | ユーザーの侵害確認 |
| コード実行の解放 | メール添付ファイルの検疫 | |
| 検疫からの解放 | 外部メール転送の無効化 | |
| サンプルの要求 | | |
| コード実行の制限（元に戻すことが可能） | | |
| ウイルス対策スキャンの実行 | | |
| 停止と検疫 | | |
| ネットワークからのデバイスの封じ込め | | |

### 自動化レベル

デバイスグループごとに自動化レベルを設定可能。

| 自動化レベル | 説明 |
|---|---|
| **Full（完全自動化）** | 悪意があると判断されたエンティティに対して修復アクションが自動実行される。**推奨設定**。 |
| **Semi - すべてのフォルダーで承認が必要** | すべてのファイルに対する修復アクションに承認が必要。保留中のアクションは 7 日でタイムアウトする。 |
| **Semi - コアフォルダーの修復に承認が必要** | OS ディレクトリ（`\windows\*`）内のファイルに対してのみ承認が必要。その他のフォルダーは自動実行。 |
| **Semi - 一時フォルダー以外の修復に承認が必要** | 一時フォルダー（`\temp\*`, `\downloads\*` 等）以外のファイルに対して承認が必要。 |
| **No automated response（自動化なし）** | 自動調査が実行されない。**非推奨**。 |

> **推奨事項**: Microsoft は完全自動化（Full）を推奨しており、完全自動化を使用している組織は、低い自動化レベルに比べて 40% 多くの高信頼度マルウェアサンプルを除去できるというデータがある。

## 2. 自動攻撃中断 (Automatic Attack Disruption)

### 概要

進行中の高度な攻撃（ランサムウェアキャンペーンなど）をリアルタイムで自動的に封じ込める機能。攻撃者が使用する侵害済みの資産を自動的に封じ込め、横方向移動（ラテラルムーブメント）を早期に制限する。

### 動作の 3 段階

1. **シグナルの相関分析**: エンドポイント、ID、メール、SaaS アプリからのシグナルを相関させ、高信頼度のインシデントを特定
2. **侵害資産の特定**: 攻撃者が制御し、攻撃拡散に使用している資産を特定
3. **自動応答アクションの実行**: 関連する Microsoft Defender 製品全体で応答アクションを自動実行

### 自動応答アクション

| アクション | 説明 |
|---|---|
| **デバイスの封じ込め (Device Contain)** | Defender for Endpoint の機能。疑わしいデバイスの受信/送信通信をブロックする自動封じ込め。未検出/未オンボードデバイスの悪意ある IP アドレスも自動封じ込めが可能。 |
| **ユーザーの無効化 (Disable User)** | Defender for Identity の機能。侵害されたアカウントを自動的に一時停止。AD ユーザー、Entra ID 同期ユーザー、クラウドネイティブアカウントに対応。 |
| **ユーザーの封じ込め (Contain User)** | Defender for Endpoint の機能。疑わしい ID を一時的に封じ込め、ラテラルムーブメントや関連する通信をブロック。 |

### 攻撃中断の識別方法

- インシデントキューに **Attack Disruption** タグが表示される
- インシデントページの上部に黄色のバナーが表示される
- インシデントグラフに資産のステータス（アカウント無効化、デバイス封じ込め等）が表示される
- API ではインシデントタイトル末尾に `(attack disruption)` が追加される

## 3. カスタム検出ルール (Custom Detection Rules)

### 概要

高度なハンティング（Advanced Hunting）クエリを使用して、独自の検出ルールを作成できる。ルールに一致するイベントが発生した際に、自動的にアラートの生成や応答アクションの実行が可能。

### 主な特徴

- **KQL（Kusto Query Language）ベース**: 高度なハンティングクエリを使用してルールを定義
- **スケジュール実行**: 12 時間、24 時間、またはカスタム間隔で定期実行
- **自動応答アクション**: ルールに一致した場合に自動で実行されるアクションを設定可能

### 設定可能な自動応答アクション

| 対象 | アクション |
|---|---|
| **デバイス** | ウイルス対策スキャンの実行、デバイスの分離、調査パッケージの収集、コード実行の制限、アプリの実行制限 |
| **ファイル** | ファイルの検疫、ファイルのブロック |
| **ユーザー** | ユーザーの侵害確認としてのマーク、ユーザーへのサインインの強制 |
| **メール** | メールの論理削除、メールの受信トレイへの移動、迷惑メールフォルダーへの移動 |

## 4. アクションセンター (Action Center)

### 概要

自動調査・手動調査に関係なく、すべての修復アクションを一元管理するダッシュボード。

### 主な機能

- **保留中のアクション（Pending tab）**: 承認待ちのアクションを表示・承認・拒否
- **履歴（History tab）**: 完了済みのアクションを表示
- **元に戻す機能**: 完了したアクションを必要に応じて元に戻すことが可能
- **統合ビュー**: Defender for Endpoint、Defender for Office 365、Defender for Identity の修復アクションを統合表示

### アクセス方法

Microsoft Defender ポータル（https://security.microsoft.com） → **アクションセンター**

---

## 前提条件

| 要件 | 詳細 |
|---|---|
| **サブスクリプション** | Microsoft 365 E5 / A5、Microsoft 365 E3 + Microsoft Defender Suite アドオン、Office 365 E5 + EMS E5 + Windows E5 のいずれか |
| **デバイス要件** | Windows 10 バージョン 1709 以降 または Windows 11、Defender for Endpoint および Defender Antivirus の構成 |
| **ネットワーク要件** | Defender for Identity の有効化、Defender for Cloud Apps の構成 |
| **メール保護** | Defender for Office 365 の構成 |
| **必要な権限** | グローバル管理者またはセキュリティ管理者（構成用）。Microsoft は最小権限の原則を推奨。 |

---

## 構成手順

1. **前提条件の確認**: ライセンス、デバイス、ネットワーク要件を満たしていることを確認
2. **デバイスグループの自動化レベル設定**: Microsoft Defender ポータル → **システム** → **設定** → **エンドポイント** → **デバイスグループ** → **修復レベル**を確認・変更
3. **Office 365 のセキュリティおよびアラートポリシーの確認**: **ポリシーとルール** → **アラートポリシー** で脅威管理カテゴリのポリシーを確認
4. **攻撃中断の構成**: 自動攻撃中断の設定を確認・有効化
5. **カスタム検出ルールの作成**: 高度なハンティングクエリを使用してカスタムルールを作成
